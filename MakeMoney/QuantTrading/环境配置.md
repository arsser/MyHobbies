# 环境配置与虚拟环境管理 (uv)

本项目使用 [uv](https://github.com/astral-sh/uv) 替代传统的 `pip` 和 `conda` 进行环境管理。`uv` 是一个基于 Rust 的超快 Python 包管理器，非常适合快速迭代的量化开发。

## 1. 为什么选择 uv？
*   **极致的速度**：安装速度比 pip 快 10-100 倍。
*   **确定性**：通过 `uv.lock` 锁定版本，确保代码在任何机器上运行结果一致。
*   **无需手动创建虚拟环境**：`uv` 会自动处理 `.venv`。

---

## 2. 核心操作流程

### 初始化环境
如果您是第一次克隆本项目，可以在项目根目录下执行：
```bash
uv sync
```
这会根据 `pyproject.toml` 自动创建 `.venv` 并安装所有依赖。

### 添加新依赖
如果您需要新的库（如回测框架 `vectorbt`）：
```bash
uv add vectorbt
```

### 运行脚本
**这是最推荐的运行方式**，它会自动使用虚拟环境中的 Python：
```bash
uv run 01-test_data_api.py
```

### 交互式环境 (REPL)
查看数据时非常有用的交互模式：
```bash
uv run python
```

## 4. 编辑器配置 (VsCode)
1.  按 `Cmd + Shift + P`。
2.  输入 `Python: Select Interpreter`。
3.  选择项目根目录下 `.venv/bin/python`。

---

## 5. 故障排除
如果环境出现混乱，可以大胆删除 `.venv` 文件夹并重新同步：
```bash
rm -rf .venv
uv sync
```
