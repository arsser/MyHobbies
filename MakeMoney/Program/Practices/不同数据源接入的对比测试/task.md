# 1. 目的
测S试不同股票数据源的性能，包括是否可连接，能否稳定的的连接。

# 2. 目标股票：
A股：招商银行(600036)
H股：腾讯(00700HK)

# 3. 候选数据源：
优先测试以下 4 类数据源（先免费后付费）：

1). 开源 Python 库（免费）
- AkShare（A/H 股覆盖较好，社区活跃）
- efinance（A 股日线/分时常用，接入简单）
- BaoStock（历史数据相对稳定，字段偏基础）

2). 官网/开放接口（免费或低成本）
- 东方财富公开接口（通过现有封装使用）
- 新浪/腾讯公开行情接口（作为补充校验源）

3). 第三方平台 API（需申请）
- Tushare Pro（积分制，字段完整）
- 聚宽/JQData（研究场景常用）

4). 券商/行情终端 API（可选，后续）
- MiniQMT/券商开放 API（用于实盘前验证）

备注：第一阶段先完成 AkShare、efinance、BaoStock 三源横向对比。

# 4. 设计方案（待完善）：
1). 测试方法：
- 测试维度：
  - 连通性：是否可正常请求并返回有效数据
  - 稳定性：连续请求成功率、失败重试后恢复率
  - 性能：单次请求耗时、批量请求总耗时、吞吐量
  - 数据质量：字段完整性、时间范围一致性、复权一致性
- 测试对象：
  - A 股：招商银行（600036）
  - H 股：腾讯（00700.HK）
- 测试场景（每个数据源都执行）：
  - S1: 单票日线拉取（近 1 年）
  - S2: 单票分钟线拉取（近 20 交易日，若支持）
  - S3: 批量股票拉取（30 支股票，日线）
  - S4: 连续稳定性压测（每 5 秒 1 次，持续 30 分钟）
- 统计指标：
  - 成功率 = 成功请求数 / 总请求数
  - 平均耗时、P95 耗时、最大耗时
  - 空值率、重复记录数、时间戳缺口数
- 异常策略：
  - 超时阈值：10 秒
  - 重试策略：指数退避，最多 3 次
  - 失败日志：记录请求参数、异常类型、错误堆栈

2). 开发语言：
- Python 3.11+
- 依赖建议：
  - 数据源 SDK：akshare、efinance、baostock
  - 数据处理：pandas、numpy
  - 统计可视化：matplotlib / seaborn
  - 稳定性工具：tenacity（重试）、loguru（日志）
- 工程结构建议：
  - `sources/`：各数据源统一适配器
  - `tests/`：场景化测试脚本（S1~S4）
  - `reports/`：输出 CSV + 图表 + markdown 结论

3). 测试报告：
- 报告结构：
  - 测试环境（系统、网络、Python 版本、依赖版本）
  - 测试范围（股票、时间区间、数据频率）
  - 核心结果（成功率、耗时、稳定性、质量评分）
  - 问题清单（超时、限流、字段缺失、异常案例）
  - 结论与建议（推荐主数据源 + 备份数据源）
- 输出文件：
  - `raw_metrics.csv`：原始请求指标
  - `summary_metrics.csv`：聚合指标
  - `report.md`：结论报告（可直接阅读）

# 5. uv 正确用法（避免常见坑）
1). 进入项目目录后再执行：
- `cd "MakeMoney/Program/Practices/不同数据源接入的对比测试"`

2). 安装依赖：
- `uv pip install -r requirements.txt`

3). 运行脚本（推荐）：
- `uv run python "./tests/run_benchmark.py"`

4). 常见错误示例（不要这样）：
- `uv python "./tests/run_benchmark.py"`
- 原因：`uv python` 后面需要的是 Python 子命令，不是脚本路径。

5). 兜底方式：
- 如果不使用 uv，也可以直接运行：`python "./tests/run_benchmark.py"`
